# Self-Improving Lane Index

Purpose: keep DemoAgent corrections and post-task reflections searchable without making them hot memory.

## Read When

- The user says DemoAgent repeated a mistake.
- A runtime task fails for a reusable workflow reason.
- A post-task review asks what should change next time.

## Files

- `corrections.md` - explicit user corrections and do-not-repeat lessons.
- `reflections.md` - reusable workflow reflections.

## Boundary

Records here are candidates. They do not change `MEMORY.md`, persona, tool routing, or framework policy unless promoted through `pending/memory-updates/` and approved.
